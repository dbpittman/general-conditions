## Submittal Schedule

<!-- Define each required submittal. Not every project needs all of these. Delete rows that don't apply. -->

| Submittal | When | Reviewer | Required Before |
|-----------|------|----------|-----------------|
| Execution Plan | Before starting work | User | Any detailed execution |
| Architecture/Approach | After plan approval | User | Component implementation |
| Implementation Plan (per major component) | Before building component | User/Evaluator | Component execution |
| Progress Report | At each milestone | User | Proceeding to next phase |
| Test Results | After implementation | Evaluator | Delivery |
| Documentation Draft | Before final delivery | User | Final delivery |
| Final Delivery Package | At completion | User | Project close |

## Submittal Format

<!-- How should submittals be delivered? -->

- **Execution Plan:** Markdown in the project repo / chat message (choose one)
- **Implementation Plans:** Markdown or PR description
- **Progress Reports:** Structured summary per GC-05 Section 5.2
- **Test Results:** Automated test output or manual test report
- **Final Package:** Per GC-12 Section 12.1
