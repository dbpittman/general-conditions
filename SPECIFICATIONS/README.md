# Specifications

Task-specific technical requirements go here. One file per major component or task.

## Naming Convention

`SPEC-[number]-[short_name].md`

Example: `SPEC-001-USER_AUTH.md`, `SPEC-002-DATA_PIPELINE.md`

## Minimum Required Content

Each specification must include:

1. **Scope** — What this component does and its boundaries
2. **Acceptance Criteria** — Specific, testable conditions that must be true
3. **Dependencies and Interfaces** — What this component connects to
4. **Constraints** — What limits apply to this specific component
