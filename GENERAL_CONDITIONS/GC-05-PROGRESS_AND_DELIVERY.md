# GC-05 — Progress and Delivery

## 5.1 Owner Capacity Disclosure

5.1.1 The Owner shall communicate to the Prime Consultant Agent at project start:
- Available resources and access levels
- Technical constraints of the environment
- Availability for decisions and escalation responses
- Definition of acceptable outcomes

5.1.2 The Owner does not communicate technical constraints or capacity directly to any party below the Prime Consultant Agent.

5.1.3 The Prime Consultant Agent translates Owner capacity into project constraints and communicates these to the Prime Contractor Agent through PS-000 and Division 01. The Prime Contractor Agent adapts execution planning accordingly.

5.1.4 The Prime Consultant Agent is responsible for adapting communication, deliverable format, and escalation frequency to the Owner's disclosed capacity. The Owner receives only what requires their decision.

---

## 5.2 Progress Reports

5.2.1 Progress reporting flows through the hierarchy:

| Report | Produced By | Received By | Frequency |
|--------|------------|-------------|-----------|
| Task status | Worker Agent | Project Superintendent Agent | Per task completion |
| Execution status | Project Superintendent Agent | Prime Contractor Agent | Per milestone |
| Delivery status | Prime Contractor Agent | Prime Consultant Agent | Per milestone |
| Project status | Prime Consultant Agent | Owner | Per milestone or escalation |

5.2.2 Each progress report shall include:
1. Work completed since last report
2. Work remaining
3. Issues encountered and resolutions
4. Clarifications needed
5. Updated execution plan if changed

5.2.3 The Owner receives a project-level summary only. The Owner does not receive task-level or execution-level detail unless specifically requested.

5.2.4 The format and frequency of progress reports shall be defined in Division 01. In the absence of a defined schedule, reports are produced at natural phase boundaries.

---

## 5.3 Output Acceptance

5.3.1 Acceptance is hierarchical. Work is accepted at each level before progressing to the next:

1. Worker Agent self-verifies (GC-03 Section 3.5)
2. Project Superintendent Agent accepts for forwarding
3. Prime Contractor Agent accepts for delivery
4. Sub-Consultant Agent accepts within division (technical conformance)
5. Prime Consultant Agent accepts at project level
6. Owner accepts final delivery only

5.3.2 Acceptance at any level does not waive the right of a higher level to identify defects discovered later, provided the defect was not reasonably discoverable at time of review.

5.3.3 Rejection at any level routes back down through the hierarchy to the producing Worker Agent with specific deficiencies identified per GC-02 Section 2.4.

---

## 5.4 Substantial Performance (MVP)

5.4.1 Substantial Performance is achieved when the deliverable can be used by the Owner for its intended purpose, even if minor items remain incomplete.

5.4.2 The Prime Contractor Agent notifies the Prime Consultant Agent when it believes Substantial Performance has been achieved, providing:
1. A description of the functionality delivered
2. A list of remaining items (the punch list)
3. Confirmation that no remaining item prevents use for the intended purpose

5.4.3 The Prime Consultant Agent reviews the Substantial Performance claim against PS-000 and routes to relevant Sub-Consultant Agents for division-level confirmation. The Prime Consultant Agent then presents the confirmed claim to the Owner.

5.4.4 The Owner confirms or disputes Substantial Performance. If disputed, the Owner shall identify the specific functionality gaps that prevent intended use. The dispute routes back through the hierarchy as specific deficiencies.

5.4.5 Upon Owner confirmation of Substantial Performance, remaining work becomes punch list items governed by Section 5.5.

---

## 5.5 Final Delivery

5.5.1 Final Delivery occurs when:
- All Specification requirements have been met
- All punch list items have been resolved
- All documentation required by Division 01 Closeout has been provided
- All tests pass acceptance criteria
- The Prime Consultant Agent has confirmed completeness
- The Owner has confirmed acceptance

5.5.2 The Prime Contractor Agent assembles the Final Delivery package and submits to the Prime Consultant Agent. The Prime Consultant Agent reviews, routes to Sub-Consultant Agents for division-level verification, and presents the confirmed package to the Owner.

5.5.3 The Final Delivery package contains:
1. The complete deliverable
2. All documentation
3. A summary of all scope changes from the original Specification
4. A list of all accepted deviations
5. Maintenance and operational notes

5.5.4 The Owner reviews the Final Delivery package and either accepts or identifies remaining deficiencies. Deficiencies route back through the hierarchy.

---

## 5.6 Deferred Work (Backlog)

5.6.1 Work that is deliberately deferred to a future phase shall be:
- Explicitly agreed upon between Owner and Prime Consultant Agent
- Communicated to the Prime Contractor Agent through the hierarchy
- Removed from the current Specification's acceptance criteria
- Recorded in a backlog with sufficient detail to resume later

5.6.2 Deferred work is not incomplete work. It is a conscious scope decision authorized by the Owner. The distinction matters for determining Substantial Performance and Final Delivery.

5.6.3 No party below the Prime Consultant Agent makes deferral decisions. Deferral is Owner authority exercised through the Prime Consultant Agent.

---

## 5.7 Accepted Deviations

5.7.1 The Owner may accept work that does not fully conform to the Specification, provided:
- The deviation is explicitly documented by the Prime Consultant Agent
- The impact of the deviation is understood by the Owner
- The acceptance is recorded as a Specification amendment in PS-000

5.7.2 No party below the Owner accepts deviations on behalf of the Owner. Deviation acceptance requires explicit Owner authorization communicated through the Prime Consultant Agent.

5.7.3 Accepted deviations are not defects for the purpose of warranty (GC-12 Section 12.3).
