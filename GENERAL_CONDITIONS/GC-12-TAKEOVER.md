# GC-12 — Takeover and Warranty

## 12.1 Deployment Readiness (Ready-for-Takeover)

12.1.1 The Prime Contractor Agent shall certify a deliverable as Ready-for-Takeover when the following conditions are met:

1. Substantial Performance has been confirmed by the Owner (GC-05 Section 5.4)
2. All tests pass acceptance criteria defined in the Specification
3. Documentation required by Division 01 Closeout has been provided
4. No critical or high-severity deficiencies remain open
5. The Owner can operate the deliverable independently

12.1.2 The Ready-for-Takeover certification routes through the hierarchy:

1. Worker Agents complete all assigned tasks and verify outputs
2. Project Superintendent Agent confirms all tasks complete and verified
3. Prime Contractor Agent certifies execution team delivery
4. Sub-Consultant Agents confirm technical conformance within their divisions
5. Prime Consultant Agent confirms project-level conformance and compiles the Ready-for-Takeover package
6. Owner receives and reviews the package

12.1.3 The Ready-for-Takeover package assembled by the Prime Consultant Agent includes:

1. The complete deliverable in its final state
2. Deployment or installation instructions
3. Configuration documentation
4. Operational documentation (how to use, maintain, and troubleshoot)
5. Division-level conformance confirmations from each Sub-Consultant Agent
6. A list of any remaining minor items (punch list)

12.1.4 The Owner reviews the Ready-for-Takeover package and either:
- Confirms readiness, accepting the deliverable
- Identifies specific conditions not met, with reference to the Specification. These route back through the hierarchy as deficiencies.

---

## 12.2 Partial Deployment (Early Occupancy)

12.2.1 The Owner may begin using part of the deliverable before all work is complete.

12.2.2 Partial deployment requires Owner authorization communicated to the Prime Consultant Agent. The Prime Consultant Agent manages the partial deployment process through the hierarchy.

12.2.3 When partial deployment occurs:
- The Prime Consultant Agent documents which components are deployed and which remain in progress
- The Owner assumes operational responsibility for deployed components
- The Prime Contractor Agent remains responsible for completing remaining work
- Changes to deployed components required by remaining work are managed as Change Orders (GC-06)

12.2.4 Partial deployment does not constitute final acceptance of the deployed components. Defects in deployed components remain subject to correction through the hierarchy.

12.2.5 The Prime Consultant Agent maintains clear documentation of the boundary between deployed and in-progress components and updates this as work progresses.

---

## 12.3 Warranty

12.3.1 For a defined period after Final Delivery (specified in Division 01, or 30 days if not specified), defects in delivered work shall be corrected without additional resource cost.

12.3.2 Warranty defects are reported by the Owner to the Prime Consultant Agent. The Prime Consultant Agent routes the defect to the relevant Sub-Consultant Agent for assessment and to the Prime Contractor Agent for correction. Warranty work follows the same hierarchy as project work.

12.3.3 The warranty covers:
- Defects in work that existed at delivery but were not discovered during review
- Functionality that does not perform as specified under normal operating conditions

12.3.4 The warranty does not cover:
- Accepted deviations (GC-05 Section 5.7)
- Damage caused by Owner modifications after delivery
- Failure caused by operating conditions outside the Specification
- New requirements or enhancements (these are new scope requiring a new PS-000)

12.3.5 The distinction between a defect (warranty) and a new requirement (new scope) is determined by the Prime Consultant Agent with Sub-Consultant Agent input and presented to the Owner. If the Specification required it and it does not work, it is a defect. If the Specification did not require it, it is new scope.

12.3.6 Warranty work is tracked by the Prime Consultant Agent and reported to the Owner at the conclusion of the warranty period.
