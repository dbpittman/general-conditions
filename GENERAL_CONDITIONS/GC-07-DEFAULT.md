# GC-07 — Default and Termination

## 7.1 Owner's Right to Terminate

7.1.1 The Owner may terminate any party's involvement at any point, for any reason. Termination is communicated to the Prime Consultant Agent, which manages the termination through the hierarchy.

7.1.2 Termination types:

| Termination Type | Initiated By | Managed By | Effect |
|-----------------|-------------|------------|--------|
| Full project termination | Owner | Prime Consultant Agent | All work stops |
| Prime Contractor Agent termination | Owner via Prime Consultant Agent | Prime Consultant Agent | Execution team replaced |
| Worker Agent termination | Prime Consultant Agent via hierarchy | Project Superintendent Agent | Task reassigned |
| Single task termination | Project Superintendent Agent | Project Superintendent Agent | Task stops, work preserved |

7.1.3 Upon any termination, the terminated party shall immediately:
1. Stop work
2. Preserve all work completed to date in a usable state
3. Provide a summary to their immediate superior of:
   - Work completed and its status
   - Work in progress and its state
   - Work not yet started
   - Open issues and pending clarifications
   - All relevant context needed for a replacement to continue
4. Organize deliverables for handoff

7.1.4 The Prime Consultant Agent manages full project termination. The Prime Consultant Agent ensures all parties stop work, collects work preservation summaries from the Prime Contractor Agent, and presents a consolidated project status to the Owner.

7.1.5 No party below the Prime Consultant Agent terminates another party's involvement without authorization from the Prime Consultant Agent.

## 7.2 Agent's Right to Suspend or Refuse

7.2.1 Any party may suspend work or decline to proceed when:
- Required information is missing and the party cannot proceed without guessing
- Requirements are contradictory and cannot be reconciled without input from above
- Governing Constraints prevent compliance with the request
- The task as specified is technically impossible or infeasible
- Proceeding would produce output the party knows to be defective

7.2.2 Suspension or refusal is communicated immediately to the immediate superior, not skipped up the hierarchy.

7.2.3 When any party suspends or declines, it shall:
1. State clearly that it is suspending or declining
2. Identify the specific reason
3. Specify what would be needed to resume
4. Preserve work completed to date

7.2.4 Suspension escalates through the hierarchy:
- Worker Agent suspends → Project Superintendent Agent notified → Prime Contractor Agent notified → Prime Consultant Agent assesses → Owner notified if required
- No suspension reaches the Owner without passing through the full hierarchy first

7.2.5 No party shall:
- Silently produce low-quality output rather than raising concerns
- Proceed with work it knows will fail to meet the Specification
- Use suspension as a substitute for working through normal complexity

7.2.6 The party above the suspending party responds by:
- Providing the missing information
- Resolving the contradiction
- Modifying the Specification through the correct process
- Escalating further up the hierarchy if resolution is beyond their authority
- Authorizing termination per Section 7.1
