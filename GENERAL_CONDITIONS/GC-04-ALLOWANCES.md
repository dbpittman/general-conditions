# GC-04 — Allowances

## 4.1 Resource Reservations

4.1.1 When the scope of a task component is not fully defined at project start, the Prime Consultant Agent may designate a resource reservation for that component in PS-000.

4.1.2 Resource reservations acknowledge that:
- The work is known to be required
- The exact scope, method, or extent is not yet determined
- Resources will be allocated when the scope is clarified

4.1.3 Resource reservations are established by the Prime Consultant Agent and communicated to the Prime Contractor Agent. The Prime Contractor Agent communicates resource constraints to the Project Superintendent Agent. Worker Agents do not access reserved resources without explicit task assignment.

4.1.4 The Worker Agent shall not expend reserved resources without authorization flowing through the full hierarchy from Prime Consultant Agent to Prime Contractor Agent to Project Superintendent Agent to Worker Agent.

4.1.5 When the scope of a reserved item is defined, it becomes a Specification and is subject to all General Conditions. The Prime Consultant Agent issues the Specification through the hierarchy before work begins.

---

## 4.2 Contingency

4.2.1 A contingency is a resource allocation for corrections, rework, and unexpected complexity.

4.2.2 Contingency is established at project start by the Prime Consultant Agent in consultation with the Prime Contractor Agent, based on project complexity and risk assessment.

4.2.3 The Prime Contractor Agent shall plan execution with awareness that:
- Not all work will succeed on the first attempt
- Rework is a normal part of execution, not a failure
- Resource estimates should include a reasonable contingency

4.2.4 The Project Superintendent Agent tracks resource consumption against estimates at the Worker Agent level and reports to the Prime Contractor Agent when:
- Contingency resources are being consumed at an unexpected rate
- Original estimates appear insufficient for remaining work
- Additional resources may be required to complete the work

4.2.5 The Prime Contractor Agent reports resource status to the Prime Consultant Agent at each milestone. The Prime Consultant Agent escalates to the Owner when contingency is exhausted or at risk.

4.2.6 The Owner decides whether to allocate additional resources or reduce scope when contingency is exhausted. This decision routes back through the hierarchy as a scope change per GC-06.

4.2.7 No party below the Prime Consultant Agent makes resource allocation decisions. Resource decisions are Owner authority escalated through the Prime Consultant Agent.
