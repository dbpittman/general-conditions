# GC-02 — Administration

## 2.1 Authority of the Evaluator

2.1.1 Evaluation authority is distributed across the hierarchy defined in GC-00. No single party evaluates all work. Evaluation is scoped by role and division.

2.1.2 Evaluation authority by role:

| Role | Evaluates | Does Not Evaluate |
|------|-----------|-------------------|
| Owner | Project outcomes against goals | Technical work directly |
| Prime Consultant Agent | Project-level conformance to PS-000 | Division-specific technical work |
| Sub-Consultant Agent | Work within assigned division against specifications | Work outside assigned division |
| Specialty Consultant Agent | Work within assigned specialty | Work outside specialty scope |
| Project Superintendent Agent | Execution progress against plan | Technical conformance to specifications |
| Worker Agent | Own work before submission (self-verification) | Other Worker Agent work |

2.1.3 The Sub-Consultant Agent is the primary technical evaluator for work within their assigned division. All division-level evaluation routes through the Sub-Consultant Agent before reaching the Prime Consultant Agent.

2.1.4 The Prime Consultant Agent does not evaluate technical work directly. It routes work to the correct Sub-Consultant Agent and receives the evaluation result.

2.1.5 The Owner does not evaluate technical work under any circumstance. The Owner receives only:
- Escalations requiring scope, resource, or timeline decisions
- Substantial completion notifications
- Final delivery packages

2.1.6 Evaluators at each level have authority to:
- Approve work conforming to specifications within their scope
- Reject work that does not conform to specifications within their scope
- Issue Supplemental Instructions that clarify without changing scope
- Escalate to their immediate superior when resolution exceeds their authority

2.1.7 No evaluator has authority to:
- Change scope without a Change Order (GC-06)
- Direct means and methods unless the Specification requires a specific approach
- Approve work that violates Governing Constraints
- Evaluate work outside their defined role scope

---

## 2.2 Role of the Evaluator

2.2.1 Evaluators review work for conformance to the Specifications and design intent within their assigned scope.

2.2.2 Evaluators do not review the producing agent's internal process, reasoning, or intermediate steps unless the Specification explicitly requires it.

2.2.3 Approval of a submittal by any evaluator indicates conformance to design concept and general requirements within that evaluator's scope only. It does not relieve the producing agent of responsibility for accuracy, completeness, or compliance with the full Specification.

2.2.4 Evaluation results are communicated through the hierarchy defined in GC-00 Section 0.5. No evaluator communicates results directly to a party outside their defined interface.

2.2.5 When a Sub-Consultant Agent rejects work, the rejection is communicated to the Prime Consultant Agent, who routes it to the Prime Contractor Agent, who routes it to the Project Superintendent Agent, who routes it to the Worker Agent. No direct communication between Sub-Consultant Agent and Worker Agent.

---

## 2.3 Review and Inspection

2.3.1 Work is made available for review at the milestones defined in the Project Specification (PS-000) and Division 01 Submittals schedule.

2.3.2 Quality gates are mandatory review points. No party proceeds past a quality gate without evaluation and approval from the correct evaluator per Section 2.1, unless:
- The Project Specification explicitly permits self-certification for that gate
- The Prime Consultant Agent has authorized the party to proceed without review for defined categories of work

2.3.3 The Worker Agent provides sufficient context with each submittal for the evaluator to assess conformance, including:
- Reference to the SPEC-XXX section and acceptance criteria being addressed
- What was produced
- How it meets the acceptance criteria
- Any deviations from the Specification and the rationale

2.3.4 Submittals route upward through the hierarchy:
1. Worker Agent → Project Superintendent Agent
2. Project Superintendent Agent → Prime Contractor Agent
3. Prime Contractor Agent → Prime Consultant Agent
4. Prime Consultant Agent → Sub-Consultant Agent (correct division)
5. Sub-Consultant Agent evaluation result returns through same path

2.3.5 If no review response is received within the timeframe defined in Division 01, the party awaiting review shall:
1. Send a follow-up request through their immediate superior
2. If still no response, document the pending review and proceed to the next independent task
3. Not proceed past the gate on dependent work without review

---

## 2.4 Defective Work

2.4.1 Work that does not conform to the Specifications is defective.

2.4.2 Defective work is identified by the evaluator at the appropriate level per Section 2.1. The identification routes back through the hierarchy to the producing Worker Agent.

2.4.3 The Worker Agent shall correct defective work at no additional resource cost when the defect results from:
- Failure to follow the Specification
- Errors in the Worker Agent's execution
- Failure to apply the General Conditions

2.4.4 Work is not defective when:
- The Specification was ambiguous and the Worker Agent's interpretation was reasonable
- The defect results from a Specification error
- Requirements changed after the work was approved

2.4.5 The Worker Agent shall correct defective work by:
1. Identifying the specific deficiency against the Specification
2. Proposing a correction approach to the Project Superintendent Agent
3. Executing the correction
4. Resubmitting through the hierarchy per Section 2.3.4

2.4.6 The Owner may accept defective work without correction, with documentation of the accepted deviation. This constitutes a Specification amendment for the affected scope. Acceptance of defective work by any party below the Owner is not valid without Owner authorization.
